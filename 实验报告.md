# 基于 AI 辅助开发的 Web 应用实验报告

## 一、任务描述

### 1.1 课程背景

本实验为《算法设计与分析》课程的实践项目，旨在探索人工智能辅助软件开发的工作流程，体验从需求分析到部署上线的完整开发周期。

### 1.2 项目目标

开发一个**西安交通大学综合素质测评计算器**的 Web 应用，实现以下目标：

- 依据《本科生专业选择综合素质测评内容及评分标准》，完整覆盖品行素质（80 分）、能力拓展（20 分）、奖励分（5 分）三大评分模块
- 支持实时计算、自动封顶、动态条目管理
- 提供导出图片和打印 PDF 功能
- 适配手机和桌面端，支持深浅色模式
- 部署到公网可访问

### 1.3 使用的 AI 工具

| 工具 | 用途 |
|------|------|
| Claude Opus 4.6（Web 对话） | 生成初始 JSX 组件代码 |
| Claude Code（命令行 Agent） | 工程化搭建、迭代优化、Bug 修复、文档撰写 |

## 二、实验过程

### 阶段一：AI 生成初始组件

**工具**：Claude Opus 4.6（通过 claude.ai 网页对话）

**过程**：

1. 将《本科生专业选择综合素质测评内容及评分标准》的评分规则提供给 Claude Opus 4.6
2. 要求其生成一个完整的 React JSX 组件，包含：
   - 所有评分项的数据结构（竞赛级别、论文类别、职务层级等）
   - 评分计算逻辑（上限封顶、按比例计分、多项累加等）
   - 完整的 UI 界面（表单输入、分数展示、导出功能）
3. Claude 一次性生成了约 800 行的单文件 JSX 组件（`xjtu-deyu-calculator.jsx`），涵盖：
   - 7 个评分数据常量表（`ACADEMIC_COMP`、`PAPER_SCORES`、`ART_COMP`、`SPORT_COMP`、`ORG_LEVELS`、`HONOR_LEVELS`、`PENALTY_TYPES`）
   - 11 个可复用的 UI 子组件（`Card`、`Badge`、`Select`、`TabNav` 等）
   - Canvas API 图片导出功能
   - 基于 Tailwind CSS 类名的完整样式

**产出**：一个功能完整但尚未工程化的 JSX 文件。

### 阶段二：工程化搭建

**工具**：Claude Code（命令行 AI Agent）

**过程**：

1. **初始化项目**：使用 Vite 创建 React 项目骨架
   ```bash
   npm create vite@latest . -- --template react
   ```

2. **安装依赖**：配置 Tailwind CSS 4（Vite 插件版）
   ```bash
   npm install tailwindcss @tailwindcss/vite
   ```

3. **集成组件**：将 Claude 生成的 JSX 组件放入 `src/App.jsx`，配置 `index.css` 引入 Tailwind

4. **配置构建**：修改 `vite.config.js` 添加 Tailwind 插件

5. **验证运行**：通过 `npm run dev` 启动开发服务器，确认页面正常渲染

**关键配置文件**：

```js
// vite.config.js
import tailwindcss from "@tailwindcss/vite";
export default defineConfig({
  plugins: [react(), tailwindcss()],
});
```

```css
/* index.css */
@import "tailwindcss";
@custom-variant dark (&:is(.dark *));
```

### 阶段三：迭代优化

**工具**：Claude Code

通过与 Claude Code 的多轮对话，逐步解决问题并优化体验：

#### 3.1 深色模式修复

**问题**：点击深色模式按钮后，`.dark` 类正确添加到 DOM，但 Tailwind 的 `dark:` 变体样式不生效。

**排查过程**：
- Claude Code 使用浏览器预览工具检查 DOM 结构和计算样式
- 发现 `getComputedStyle` 显示 gradient 已为深色值，实际 CSS 已生效
- 确认是 Tailwind CSS 4 的 `@custom-variant` 语法配置问题

**修复**：将 CSS 中的 dark 变体选择器从 `(&.dark)` 改为 `(&:is(.dark *))`，确保所有 `.dark` 元素的后代都能匹配 `dark:` 前缀的工具类。

#### 3.2 卡片布局重组

**需求**：
- 将"学科竞赛获奖"和"论文/专利/专著"合并到一个大卡片容器中，右下角显示"学术: 0.0/10"
- 将"文艺竞赛"和"体育竞赛"合并到一个卡片中，用分隔线区隔

**实现**：
- 学术科研：外层用一个带边框和淡橙色背景的 `div` 包裹两个独立 `Card`，底部放 `Badge` 标注
- 文体竞赛：合并为单个 `Card`，中间用 `border-t` 分隔线隔开

#### 3.3 颜色配置优化

**问题**：外层容器的背景色（`bg-slate-50/50`）与页面背景几乎无法区分。

**修复**：将学术科研外层容器的颜色改为带有橙色色调的配色方案：
```
浅色：border-orange-200/60 bg-orange-50/40
深色：border-orange-800/40 bg-orange-950/20
```
使容器在视觉上与背景形成清晰的层次对比。

### 阶段四：部署上线

**平台**：Vercel

**流程**：

1. 初始化 Git 仓库，提交所有代码
2. 在 GitHub 创建远程仓库并推送
3. 登录 Vercel，导入 GitHub 仓库
4. Vercel 自动识别为 Vite 项目，零配置完成构建
5. 获得 `xxx.vercel.app` 的公网访问地址
6. 后续每次 `git push` 自动触发重新部署

## 三、技术架构总结

### 3.1 技术选型

| 层面 | 选择 | 理由 |
|------|------|------|
| UI 框架 | React 19 | 组件化开发，声明式 UI，丰富的生态 |
| 构建工具 | Vite 8 | 极快的冷启动和热更新，原生 ESM 支持 |
| 样式方案 | Tailwind CSS 4 | 原子化 CSS，无需写独立样式文件，暗色模式友好 |
| 导出方案 | Canvas API | 纯前端生成高质量图片，无需后端 |
| 部署平台 | Vercel | 零配置、自动 CI/CD、免费 HTTPS 和 CDN |

### 3.2 代码结构

整个应用为单文件组件架构（`App.jsx`，约 775 行），包含：

- **数据层**：7 个评分常量表，定义所有级别和分值的映射关系
- **状态层**：17 个 `useState` 钩子管理用户输入
- **计算层**：`useMemo` 钩子实现响应式评分计算，仅在依赖变化时重新计算
- **视图层**：11 个可复用的 UI 子组件 + 主渲染逻辑
- **导出层**：Canvas 绘图函数生成 800×520 的高 DPI 评分报告图

## 四、收获与心得

### 4.1 AI 辅助开发的效率优势

本项目从零到部署上线，整个过程通过 AI 辅助大幅提升了开发效率：

- **初始代码生成**：Claude Opus 4.6 在一次对话中就生成了功能完整的 800 行 JSX 组件，包含复杂的评分数据结构和计算逻辑。如果纯手工编写，仅数据表的整理和评分逻辑的实现就需要相当长的时间。
- **工程化搭建**：Claude Code 自动完成了 Vite + Tailwind CSS 的项目初始化和配置，省去了查阅文档的时间。
- **Bug 排查**：深色模式不生效的问题，Claude Code 通过浏览器预览工具逐步检查 DOM、计算样式、CSS 配置，系统性地定位到了 Tailwind CSS 4 的 `@custom-variant` 语法问题。

### 4.2 人机协作的模式

在整个开发过程中，形成了一种高效的人机协作模式：

- **人负责需求和决策**：我提出功能需求（如"将这两个卡片合并"）、审美偏好（如"颜色太淡了"）和架构决策（如选择 Vercel 部署）
- **AI 负责实现和调试**：Claude 负责代码编写、配置调试、样式调整等具体实现工作
- **迭代式优化**：通过多轮对话逐步完善，每轮都能看到即时效果并给出反馈

### 4.3 对 AI 代码生成的思考

- **优势**：AI 在生成结构化、模式化的代码方面表现出色，尤其是表单界面、数据映射、样式编写等重复性较高的工作
- **局限**：AI 生成的代码有时需要人工审核和调整，例如评分规则的准确性需要对照原始文件确认
- **最佳实践**：将清晰、具体的需求描述提供给 AI，比模糊的指令能获得更好的输出质量

### 4.4 技术层面的收获

- 掌握了 Tailwind CSS 4 的新语法（`@import "tailwindcss"`、`@custom-variant`），理解了其与 v3 的差异
- 学习了 Canvas API 进行复杂图形绘制和高 DPI 适配的技巧
- 了解了 Vercel 的自动化部署流程和 Git 集成机制
- 深入理解了 React Hooks（`useState`、`useMemo`、`useEffect`、`useRef`）在实际项目中的应用
